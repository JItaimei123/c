# 浮点数在内存中的存储

IEEE 754 标准：任意一个二进制浮点数 V 可以表示为：

V = (−1)^S * M * 2^E

- S：符号位，0为正，1为负
- M：有效数字，1 ≤ M < 2
- E：指数位

## 存储格式

float（32位）：1位S + 8位E + 23位M
double（64位）：1位S + 11位E + 52位M

## 浮点数存的过程

- M：保存时舍去第一位1，只保存小数部分，读取时再加回去（节省1位有效数字）
- E：为无符号整数，存入时 E_stored = E_true + bias（float bias=127，double bias=1023）

## 浮点数取的过程

### E不全为0或不全为1（常规情况）
E_true = E_stored - bias，M 前加上第一位的1。

### E全为0
E_true = 1 - bias，M 不再加1，还原为 0.xxxxxx。表示 ±0 和接近于0的数。

### E全为1
如果 M 全为0，表示 ±∞；否则表示 NaN。

## 中间媒介 E_stored 的作用

|作用|说明|
|-|-|
|消除负号|把可能为负的 E_true 映射成非负的 E_stored|
|指数可直接比较|硬件直接把 E_stored 当无符号整数比较|
|预留特殊编码|E_stored=0和全1留给0、非规格化数、无穷大、NaN|

## 为什么浮点数无法精确保存

许多十进制小数在二进制中是无限循环的（如 0.1），而 float/double 的尾数位有限，只能截断存储近似值。

### 什么数可以精确表示？
分母（最简分数）必须是 2 的幂。如 0.5、0.25、0.75 等。

## 例题分析

```C
int n = 9;
float *pFloat = (float *)&n;
printf("n的值为：%d\n", n);     // 9
printf("*pFloat的值为：%f\n", *pFloat); // 0.000000（E全为0，非规格化数）

*pFloat = 9.0;
printf("n的值为：%d\n", n);     // 1091567616（将浮点数的二进制当作整数打印）
printf("*pFloat的值为：%f\n", *pFloat); // 9.000000
```
