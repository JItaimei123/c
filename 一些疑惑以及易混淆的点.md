# 八、一些疑惑以及易混淆的点

## 一、变量定义

```c
int a[5][5];      // a 是 5 行 5 列的二维数组
int (*p)[4];      // p 是一个指针，指向"包含 4 个 int 的一维数组"
p = a;            // 让 p 也指向数组 a 的开头
```

## 二、数组名 `a` 的退化

- `a` 出现在赋值表达式中（不在 `sizeof` 或 `&` 后面），所以**退化成指针**。
- `a` 退化的**类型**是 `int (*)[5]`，指向**第一行（5个int）**。
- `p = a;` 使得 `p` 的**数值**等于 `a` 的起始地址，但**类型不同**：最大的区别在于p和a的步长不同
- `a`（退化后）：`int (*)[5]`，步长 = `5 × sizeof(int)`
- `p`：`int (*)[4]`，步长 = `4 × sizeof(int)`

## 三、`p` 指向什么？

- 从**数值**上：`p` 指向 `a[0][0]` 的地址。
- 从**类型**上：`p` 指向 `a[0]`（第一行这个一维数组）。
- 但 `p` 的类型是 `int (*)[4]`，所以**编译器认为每行只有 4 个元素**。

## 四、`p[4][2]` 的地址计算

**`p[i][j]`** 等价于 **`*(*(p + i) + j)`**。

> **这也是为什么下标要从0开始标,因为p[i] = *(p+i), i即是数组的偏移量**

[i]代表着**行步长**的移动，[j]代表着**列步长**的移动
- **行步长** = **`4 × sizeof(int)`**（例如 16 字节）
- **列步长** = **`1 × sizeof(int)`**（例如 4 字节）

对于 `&p[4][2]`：

| 步骤 | 操作 | 移动量 |
|-|-|-|
| 1 | `p + 4` | 移动 **4 行**，每行 4 个 `int` → `4 × 4 = 16` 个 `int` |
| 2 | `*(p + 4) + 2` | 在同一行内再移动 **2 列** → `+2` 个 `int` |
| **合计** | | **`16 + 2 = 18` 个 `int`** |

所以 `&p[4][2]` 是 `(int*)p + 18`。

## 五、`&a[4][2]` 的地址计算

`a` 的类型是 `int [5][5]`，`a[4][2]` 是实际数组的第 4 行第 2 列。
- 移动 **4 行**，每行 5 个 `int` → `4 × 5 = 20` 个 `int`
- 再移动 **2 列** → `+2` 个 `int`
- **合计：`20 + 2 = 22` 个 `int`**

所以 `&a[4][2]` 是 `(int*)a + 22`。

## 关于 `char *str`

为什么 `char *str` 就代表着保存原始字符串的起始位置？

字符串类型 `char* str` 实际上就是一个指针，它指向字符数组(str)的第一个元素（即字符串的起始位置）。

```c
char str[] = "hello";
```
- `str` 的类型是 `char*`，它指向字符序列中的第一个字符 `'h'`。
- 在函数参数列表中，写 `char *str` 和 `char str[]` 是等价的。
