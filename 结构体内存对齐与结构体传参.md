# 结构体内存对齐与结构体传参

## 一、对齐规则

1. 第一个成员对齐到偏移量为0的地址处
2. 其他成员对齐到**对齐数**的整数倍地址处
   - 对齐数 = min(编译器默认对齐数, 成员变量大小)
   - VS默认对齐数为8，Linux gcc没有默认对齐数（对齐数=成员自身大小）
3. 结构体总大小为**最大对齐数**的整数倍
4. 嵌套结构体：嵌套的结构体成员对齐到自身成员中最大对齐数的整数倍处

## 分析代码1

```C
struct S1 { char c1; int i; char c2; };
// c1 偏移0，i 从偏移4开始（4字节对齐），c2 偏移8
// 总大小 = 12（最大对齐数4，9向上取整到12）
```

## 分析代码2

```C
struct S2 { char c1; char c2; int i; };
// c1 偏移0，c2 偏移1，i 从偏移4开始
// 总大小 = 8
```

**总结**：让占用空间小的成员尽量集中在一起。

## 分析代码3

```C
struct S3 { double d; char c; int i; };
// d 偏移0~7，c 偏移8，i 从偏移12开始（4对齐）
// 总大小 = 16
```

## 分析代码4（嵌套结构体）

```C
struct S4 { char c1; struct S3 s3; double d; };
// c1 偏移0，s3 从偏移8开始（8对齐，16字节），d 从偏移24开始（8字节）
// 总大小 = 32
```

## 二、为什么存在内存对齐？

1. **平台原因**：某些硬件平台只能在特定地址处取特定类型的数据
2. **性能原因**：对齐的内存只需一次访问，未对齐需两次

本质：**用空间换效率**。

## 三、结构体传参

```C
struct S { int data[1000]; int num; };
void print1(struct S s) { }  // 传值，复制4000字节 ❌
void print2(const struct S* ps) { } // 传址，复制8字节 ✅
```

|调用方式|内存复制量|能否修改原数据|
|-|-|-|
|传值调用|`sizeof(结构体)`|❌ 不能|
|传址调用|`sizeof(指针)`（4或8字节）|✅ 能|

**推荐传址调用**，即使只读也建议传指针加 `const` 保护。
