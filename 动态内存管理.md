# 动态内存管理

## 为什么要有动态内存分配

传统方式（如 `int val = 20; char arr[10] = {0};`）空间大小固定，数组长度需在编译时确定。动态内存分配让程序在运行时灵活申请和释放内存。

## malloc 和 free

### malloc

```C
void* malloc(size_t size);
```

向堆区申请一块连续可用空间，返回起始地址。
- 成功返回起始地址，失败返回 NULL
- 返回值类型为 `void*`，使用时需强制转换

```C
int* p = (int*)malloc(20);
if (p == NULL) { perror("use malloc"); return 1; }
for (int i = 0; i < 5; i++) { p[i] = i + 1; }
```

### free

```C
void free(void* ptr);
```

释放动态开辟的内存。
- ptr 指向的空间必须是动态开辟的
- ptr 为 NULL 时函数什么都不做
- free 后 ptr 变成悬空指针，需置为 NULL

```C
free(ptr);
ptr = NULL;
```

## calloc 和 realloc

### calloc

```C
void* calloc(size_t num, size_t size);
```

`calloc = malloc + memset`，额外将每个字节初始化为 0。

```C
int* p = (int*)calloc(10, sizeof(int));
```

### realloc

调整动态内存大小。可能原地扩展，也可能移动数据到新地址。

```C
int* ptr = (int*)realloc(p, 10 * sizeof(int));
if (ptr == NULL) { free(p); return 1; }
p = ptr;
```

**安全用法**：用新指针接收 realloc 返回值，不要直接 `p = realloc(p, new_size)`（失败会丢失原指针）。

## 常见动态内存错误

1. **对 NULL 解引用**：malloc 后必须检查返回值
2. **越界访问**：`for (i = 0; i <= 10; i++) *(p + i) = i;`（当 i=10 时越界）
3. **对非动态内存 free**：`int a = 10; free(&a);`
4. **释放动态内存的一部分**：p++ 后 free(p)
5. **重复释放**：两次 free 同一块内存
6. **内存泄漏**：忘记 free

## 经典笔试题

### 题目1：传值调用问题

```C
void GetMemory(char *p) { p = (char *)malloc(100); } // ❌ 修改的是形参
void Test(void) {
    char *str = NULL;
    GetMemory(str);
    strcpy(str, "hello world"); // str 仍为 NULL，崩溃
}
```

修正：传二级指针或返回值。

### 题目2：返回局部变量地址

```C
char *GetMemory(void) {
    char p[] = "hello world"; // 局部数组
    return p; // 返回栈地址，函数结束即失效
}
```

修正：用 malloc 或 static。

### 题目3：忘记检查返回值且未释放

```C
void GetMemory(char **p, int num) {
    *p = (char *)malloc(num);
}
void Test(void) {
    char *str = NULL;
    GetMemory(&str, 100);
    strcpy(str, "hello");
    // 缺少 free(str);
}
```

### 题目4：释放后继续使用

```C
free(str);
if(str != NULL) { // str 是悬空指针，但不为 NULL
    strcpy(str, "world"); // ❌ 未定义行为
}
```

## 柔性数组

结构体的最后一个成员可以是未指定大小的数组：

```C
typedef struct {
    int i;
    int a[]; // 柔性数组
} type_a;

type_a *p = (type_a*)malloc(sizeof(type_a) + 100 * sizeof(int));
```

- 必须是最后一个成员
- `sizeof` 不包含柔性数组
- 一次 malloc 一次 free，内存连续，减少碎片

## 程序内存区域划分

- **栈区**：局部变量、函数参数、返回地址（效率高，容量有限）
- **堆区**：动态内存分配（malloc/free）
- **数据段（静态区）**：全局变量、静态变量
- **代码段**：函数体的二进制代码
